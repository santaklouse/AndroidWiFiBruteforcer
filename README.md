# WiFiBruteforcer Lab

Лабораторное Android-приложение для периодического сканирования Wi-Fi и последовательной проверки десяти паролей на управляемом тестовом устройстве.

## Требования

- Android 7.0 или новее.
- Устройство должно поддерживать Android managed work profile.
- На устройстве не должно быть другого рабочего профиля.
- На устройстве должны быть включены Wi-Fi и геолокация.
- Для сборки используется JDK 17.

## Сборка

```bash
export JAVA_HOME=$(/usr/libexec/java_home -v 17)
./gradlew clean assembleDebug
```

APK будет создан в `app/build/outputs/apk/debug/app-debug.apk`.

## Установка и создание рабочего профиля

Заводской сброс не требуется. Установите APK:

```bash
adb install -r app/build/outputs/apk/debug/app-debug.apk
```

Дальнейшая настройка выполняется на устройстве:

1. Откройте установленное приложение в личном профиле.
2. Нажмите «Создать рабочий профиль».
3. Подтвердите системное создание профиля Wi-Fi Lab.
4. В списке приложений откройте WiFiBruteforcer со значком портфеля.
5. Выдайте разрешения «Устройства поблизости» и «Точная геолокация».
6. Включите системную геолокацию и нажмите «Запустить сканирование».

Экземпляр приложения в личном профиле используется только для provisioning. Wi-Fi-сервис запускается исключительно в рабочем профиле, где приложение является Profile Owner.

## Список паролей

Десять проверяемых значений находятся в `app/src/main/java/com/santaklouse/example/wifibruteforcer/LabPasswords.kt`.

## Ограничения

- Проверяются WPA/WPA2 Personal, WPA3 SAE и WEP.
- Enterprise/EAP и OWE отображаются, но не проверяются этим словарём.
- Android ограничивает фоновые Wi-Fi-сканирования. Приложение также слушает системные результаты и инициирует собственное сканирование раз в 30 минут.
- Один пароль считается подходящим только после состояния supplicant `COMPLETED` для созданного `networkId` и целевого BSSID.
